### Step 3：Claude Projects推理流程

#### 🔧 **推理Prompt设计**



markdown

```markdown
# 产品/商业模式分析任务

## 背景信息
[粘贴Step 1收集的30个问题的答案]

## 分析要求

### Phase 1：归因分析（避免被故事忽悠）
1. 这个产品声称的成功原因是什么？
2. 用归因分析检查：是否有因果倒置/可见当关键/故事当公式？
3. 真正的原因可能是什么？（天时/地利/人和的比例）

### Phase 2：天时地利人和打分
按照DOC-S048的评分标准：
- 天时（30%）：政策25% + 技术25% + 需求30% + 资本20%
- 地利（30%）：供应链25% + 牌照20% + 渠道20% + 数据20% + 用户15%
- 人和（40%）：创始人30% + 团队25% + 决策20% + 文化15% + 人才10%

每项打分0-10，给出理由。

### Phase 3：替代函数分析
1. 对标谁？（标杆的核心能力）
2. 我们缺什么？（约束条件）
3. 如何替代？（3个创新点，0-10分评估）

### Phase 4：多维度框架检验
用以下框架交叉验证：
1. 第一性原理：拆到最底层，有没有10倍降本/提效的空间？
2. 破坏式创新：是否从边缘市场切入？向上破坏的路径清晰吗？
3. 价值链创新：在用户/分发/供给/基础设施哪个环节改写规则？
4. 飞轮效应：有没有正反馈循环？
5. 网络效应：用户越多越强吗？

### Phase 5：风险与机会
1. 反事实推演（5个"如果"）：
   - 如果政策收紧？
   - 如果竞品降价50%？
   - 如果关键员工离职？
   - 如果大厂进入？
   - 如果用户需求变化？

2. 同期对照：失败竞品为什么失败？我们避免了吗？
3. 跨期对照：5年前/5年后做，结果会怎样？

## 输出要求
1. 综合评分（0-10分）+ 投资决策建议
2. 3个最大机会 + 3个最大风险
3. 如果投/做，P0行动清单（1-3条）
4. 如果不投/不做，核心理由（1-3条）
```

## 💡 **关键成功因素（3个）**

### 1. 信息收集要全面

- ✅ 30个问题覆盖5个维度
- ✅ 既要定量数据，也要定性判断
- ✅ 既要官方叙事，也要非官方信息

### 2. 理论应用要系统

- ✅ 用6个框架交叉验证（避免单一视角）
- ✅ 每个结论都注明理论依据
- ✅ 理论与事实结合（不空谈）

### 3. 输出成果要可落地

- ✅ 4份文档各有侧重
- ✅ 每个建议都有P0行动清单
- ✅ 假设可验证、可迭代

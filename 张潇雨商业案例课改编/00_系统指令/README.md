# 00_系统指令（提示词与工作流）

本目录存放本项目的“操作系统”——**提示词、协作流程、格式规范**。  
原则：**Claude（或同类模型）负责分析推理 → Cursor 负责生成/落盘文档 → 索引/追踪器保证可检索与可复利**。

---

## 文件清单与用途

- `Claude-案例拆解提示词.md`
  - 用途：拆解单个案例（快速/深度/对比三模式）
  - 产物：可直接落盘为“案例文档”的结构化要点
- `Claude-模型提炼提示词.md`
  - 用途：从多个案例里提炼可复用模型（机制、变量、边界、可证伪、最小验证）
  - 产物：模型文档（可被多个案例引用）
- `Claude-洞察归纳提示词.md`
  - 用途：跨案例归纳洞察（主题聚类→对比→机制句→可证伪→验证计划）
  - 产物：洞察文档（含反例与验证计划）
- `Cursor-文档生成指令.md`
  - 用途：把 Claude 的要点转成 Markdown，并同步更新索引/追踪器（口径统一为通用迁移）

---

## 标准协作流程（建议）

### 你提供输入（材料包）

建议按以下结构给模型（越结构化越省 token）：

- 案例名称/企业/时间范围
- 课程片段要点（带时间戳更好）
- 你认为的关键决策点（3-7 个）
- 关键数据（收入/用户/成本/份额等，允许未知但要标注）
- 你想重点回答的问题（例如：它为什么赢？为什么在这个时点转型？）

### Claude 输出（分析要点）

- 只输出**结构化要点**（不要写成完整长文）
- 必须包含：标签、时间线、关键决策、因果链、边界/假设、可迁移原则
- 明确“哪些是课程事实、哪些是推断、哪些需要补证据”

### Cursor 落盘（写入仓库）

1. 新建案例/模型/洞察文件
2. 更新对应索引（案例/模型/洞察）
3. 更新学习追踪器（开始/完成状态）
4. 可选：补充原著笔记引用（把来源和证据链补齐）

## 使用场景对照表

| 场景 | 用哪份指令 | 输出落在哪 | 必更索引 |
|---|---|---|---|
| 第一次接触一个案例，先建框架 | 案例拆解（快速） | `01_案例库/` | 案例总索引 |
| 重点案例深挖（决策/执行/数据） | 案例拆解（深度） | `01_案例库/` | 案例总索引 + 学习追踪 |
| 2-3 个相近案例找规律 | 案例拆解（对比）+ 洞察归纳 | `03_洞察库/` | 洞察总索引 |
| 发现“可复用结构”想抽成模型 | 模型提炼 | `02_模型库/` | 模型总索引 |

---

## 新手入门建议（避免踩坑）

- 不要写“课程复述笔记”，要写“决策逻辑与因果链”。
- 不要用事后诸葛亮；必须还原当时约束与信息条件。
- 每个结论都标注：**事实 / 推断 / 待验证**。
- 每个产物都必须有“适用边界”，否则不可复用。

## 🔄 工作流设计

### Phase 1：内容准备
1. **选择原文**：从dedao文件夹选择目标案例
2. **阅读整理**：提取关键信息、时间线、数据点
3. **确定拆解模式**：快速/深度/对比

### Phase 2：Claude分析
1. **复制提示词**：根据拆解模式选择对应提示词
2. **喂入材料**：原文 + 上下文信息
3. **生成结构化输出**：标注证据类型（事实/推断/待验证）

### Phase 3：文档生成
1. **Cursor生成**：按标准格式创建案例文档
2. **位置确定**：放至对应分类目录
3. **索引更新**：更新案例总索引

### Phase 4：质量检查
1. **自检清单**：按提示词要求逐项检查
2. **模型提炼**：判断是否需要抽象模型
3. **洞察关联**：检查是否触发新洞察

## 🎯 质量标准

### 案例文档标准
- [ ] **时空还原**：准确的时间线、行业背景、企业状态
- [ ] **决策拆解**：清晰的逻辑链条、权衡因素、结果评估
- [ ] **证据标注**：关键结论标注事实/推断/待验证
- [ ] **可迁移原则**：给出边界条件和验证动作

### 模型文档标准
- [ ] **因果表达**：输入→过程→输出
- [ ] **变量识别**：关键影响因素和方向性
- [ ] **适用边界**：使用条件和限制因素
- [ ] **验证机制**：如何测试模型有效性

### 洞察文档标准
- [ ] **跨案例支撑**：基于多个案例的规律
- [ ] **可证伪性**：明确的反例和边界
- [ ] **应用价值**：具体的决策启发

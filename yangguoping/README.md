### 关键概念最小索引（用来“抓住主线”）

- **根本选择**：认知内共生 vs 认知外骨骼  
- **检验法则**：拿掉AI之后，人是否更强（反脆弱性检验）  
- **核心桥梁**：专家智慧AI化执行（重建专家智慧涌现的“认知场域”，而非模仿输出）  
- **三层结构**：问题诊断（隐形代码/路径依赖）→ 个体递归（认知复利、史官系统）→ 群体飞轮（模因传染、网络相变）  
- **人机分工**：人负责异常探测/价值判断/整合；AI负责具体化/多角度/验证/长期记忆

### 如何持续更新（建议）

- **新增洞察**：优先写入 `杨国平思考更新.md`（作为"增量日志/验证路线"）
- **框架稳定后再压缩**：把新增内容提炼进 `核心框架 v1.1` 的结构（保持其"5分钟索引"属性）
- **人物画像变化**：如果认知习惯/偏好/风险点发生变化，再更新 `杨国平认知画像.md`
- **全景档案**：当新增内容足以改变叙事主线，再回填到 `杨国平思想体系.md`
- **项目计划**：每月底根据实际执行情况更新贷查查项目计划，旧版本归档到 `archive/`

# Cursor-文档生成指令（推荐版）

> **用途**：把 Claude 的结构化要点生成标准化 Markdown 文档并落盘，同时同步更新索引/追踪器  
> **原则**：Cursor 负责“工程化落盘”，不重新发明分析结论

---

## 生成规则（强制）

### 1) 文件命名

- 案例：`<企业名>-<案例关键词>.md`
- 模型：`<模型名称>.md`
- 洞察：`洞察XX-<标题>.md`（或趋势：`趋势XX-<标题>.md`）
- 月度总结：`YYYY-MM月度总结.md`

### 2) 元信息必填

每个文档顶部必须包含：
- 标题（H1）
- 日期（拆解/创建/提出）
- 来源（可追溯）
- 标签（如适用）

### 3) 口径统一

- 所有“应用/启发”统一写作：**通用启发 / 可迁移原则**（带边界与最小验证动作）

---

**生命周期**：创业期/成长期/成熟期/转型期

## 同步更新清单（每次生成后必须执行）

### 新增/更新案例
- [ ] 更新 `01_案例库/案例总索引.md`
- [ ] 更新 `05_学习记录/学习进度追踪器.md`
- [ ] 如引用新模型：创建模型文件并更新 `02_模型库/模型总索引.md`
- [ ] 如产生洞察：创建洞察文件并更新 `03_洞察库/洞察总索引.md`

### 新增/更新模型
- [ ] 更新 `02_模型库/模型总索引.md`
- [ ] 在相关案例中补充“模型标签/相关链接”

### 新增/更新洞察
- [ ] 更新 `03_洞察库/洞察总索引.md`
- [ ] 在支撑案例中补充“洞察引用/相关链接”

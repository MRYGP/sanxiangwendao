### 方案1：修复Commit Message乱码（可选）

如果需要修复已提交的commit message，可以使用交互式rebase：

```bash
# 修复最近4个commit
git rebase -i HEAD~4

# 在编辑器中，将需要修改的commit前的'pick'改为'reword'
# 保存后，Git会逐个提示你修改commit message

# 或者逐个修复：
git commit --amend -m "chore: 删除临时重组脚本"
git rebase -i HEAD~2  # 继续修复前一个
```

**注意**：如果已经push到远程，修改commit message需要force push，可能影响其他协作者。

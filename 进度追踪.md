## 🔄 工作模式说明

### 三方协作闭环

```
用户（松阳） → Claude（分析+指令） → Cursor（执行） → GitHub（同步）
     ↑_________________________________________________|
```

### 工作流程

1. **用户提建议/需求**
   - 提出任务或问题
   - 提供背景和期望

2. **Claude查看、分析、思考、推理**
   - 查看相关文档和进度追踪
   - 分析问题本质
   - 思考解决方案
   - 推理最佳路径

3. **Claude发指令给Cursor**
   - 使用"📋 Cursor执行指令 #XXX"格式
   - 包含明确的步骤和预期产出
   - 提供参考模板和标准

4. **Cursor执行修订**
   - 按照指令执行文件修改
   - 验证修改结果
   - 更新进度追踪

5. **GitHub同步验证**
   - 提交并推送到GitHub
   - 验证变更效果
   - 完成闭环

### 角色分工

- **用户（松阳）**：提出需求，最终决策
- **Claude**：分析思考，设计指令
- **Cursor**：执行修订，验证结果
- **GitHub**：同步存储，版本管理

### 关键原则

- **Claude主动**：对话开始主动检查进度追踪
- **指令清晰**：给Cursor的指令要明确、可执行
- **闭环验证**：每次GitHub更新后主动检查
- **进度更新**：任务完成及时更新进度追踪

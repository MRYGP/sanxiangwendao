## 🎯 快速决策树

### 第一步：判断分类

```
这篇文档主要解决什么问题？
    ↓
┌────────────────────┬────────────────────┐
│ 改变认知/建立框架   │ 解决具体问题       │
│ （WHY）            │ （HOW）            │
↓                    ↓
道层                  术层
│                    │
↓                    ↓
选择子分类            选择子分类
```

**道层（3类）**：
- **Theory**：建立新的认知框架（如：认知内共生理论）
- **Framework**：整合多个理论的系统框架（如：思想体系核心框架）
- **Philosophy**：人生意义与价值观（如：构建北极星）

**术层（6类）**：
- **Innovation**：创新、创业、商业模式
- **Communication**：沟通、影响力、说服
- **Behavior-Change**：习惯养成、行为改变
- **Strategy**：战略决策、系统思维
- **Execution**：执行力、效率提升
- **Psychology**：认知偏差、心理机制

## ✍️ 7步写作法（必须完成）

### 1. 故事/案例开头（200-500字）
- [ ] 真实案例（公司/个人/历史事件）
- [ ] 有冲突或反转
- [ ] 与主题直接相关

**模板**：
```markdown
## 一、[故事标题]

[具体场景描述]

**问题/冲突**：[描述问题]

**结果**：[出人意料的结果]

**这就是本文要讨论的：[核心主题]**
```

---

### 2. 核心洞察/问题
- [ ] 一句话点明本质
- [ ] 挑战常识或直觉
- [ ] 提出清晰问题

**模板**：
```markdown
## 二、核心问题：[问题陈述]

### 传统观点认为：[常识]

### 但真相是：[反常识洞察]

### 这引出一个关键问题：[核心问题]
```

---

### 3. 理论框架/方法
- [ ] 有清晰的框架图示或表格
- [ ] 有2-5个核心要素
- [ ] 有每个要素的详细解释

**模板**：
```markdown
## 三、[框架名称]：核心要素

### 框架概览

[框架图示或表格]

### 要素一：[名称]
**定义**：[清晰定义]
**为什么重要**：[价值说明]
**如何应用**：[具体方法]
```

---

### 4. 案例分析/对比
- [ ] 至少3个案例
- [ ] 有成功和失败对比
- [ ] 有数据支撑

**模板**：
```markdown
## 四、案例分析

### 案例1：[成功案例]
**背景**：[公司/场景]
**做对了什么**：[关键决策]
**结果**：[量化结果]
**启示**：[可迁移的经验]

### 案例2：[失败案例]
**背景**：[公司/场景]
**做错了什么**：[致命错误]
**结果**：[惨痛代价]
**启示**：[避坑指南]
```

---

### 5. 实践建议/行动
- [ ] 分"立即行动"和"长期坚持"
- [ ] 每条建议具体可执行
- [ ] 有明确的时间、对象、产出

**模板**：
```markdown
## 五、实践建议

### 立即行动（本周）
1. **[行动1]**：
   - 时间：[具体时间]
   - 对象：[具体对象]
   - 产出：[具体成果]

### 长期坚持（本月/季度）
1. **[战略1]**：
   - 目标：[清晰目标]
   - 方法：[具体方法]
   - 检查点：[如何验证]

### 自检问题
- [ ] [关键问题1]
- [ ] [关键问题2]
```

---

### 6. 核心洞察金句
- [ ] 至少3条金句
- [ ] 每条10-30字
- [ ] 可独立引用

**模板**：
```markdown
## 六、核心洞察

### 金句
1. **"[金句1]"**
2. **"[金句2]"**
3. **"[金句3]"**

### 核心框架
**[框架名称] = [要素1] + [要素2] + [要素3]**
```

---

### 7. 相关文档引用
- [ ] 引用至少2篇相关文档
- [ ] 说明关联关系

**模板**：
```markdown
---

*与本文相关的文档：*
- *《[文档名]》——[关系说明]*
- *《[文档名]》——[关系说明]*
```

## 🎨 写作风格检查

| 维度 | 要求 | 检查 |
|------|------|------|
| **简洁** | 每句话<40字 | [ ] |
| **具体** | 用数据和案例 | [ ] |
| **口语化** | 像说话一样 | [ ] |
| **有画面感** | 用比喻和故事 | [ ] |
| **语气** | 像带徒弟的CEO | [ ] |

**禁止**：
- [ ] 学术八股文
- [ ] 鸡汤式空洞
- [ ] 概念堆砌
- [ ] 过度修饰

## 📑 YAML索引模板

### 文件位置
```
rag-index/indexes/DOC-[D/S][编号].yaml
```

### 编号规则
- 道层：DOC-D013, DOC-D014...
- 术层：DOC-S040, DOC-S041...

### 完整模板
```yaml
---
doc_id: "DOC-[D/S][编号]"
doc_type: "[theory/methodology/framework/technique/philosophy]"
title: "[文档标题]"
summary: "[文档摘要，100-200字]"
keywords:
  - "[关键词1]"
  - "[关键词2]"
  - "[关键词3]"
  - "[关键词4]"
  - "[关键词5]"
related_docs:
  - "[相关文档ID1]"
  - "[相关文档ID2]"
theory_chain: "[理论链，如innovation_methodology]"
query_patterns:
  - "[查询模式1]"
  - "[查询模式2]"
  - "[查询模式3]"
doc_weight: "[core/important/reference]"
layer: "[dao/shu]"
training_order: [培训顺序，如40]
training_questions:
  - "[培训问题1]"
  - "[培训问题2]"
---
```

### 字段说明速查

| 字段 | 说明 | 示例 |
|------|------|------|
| **doc_id** | 文档唯一标识 | DOC-S040 |
| **doc_type** | 文档类型 | methodology |
| **title** | 文档标题 | 规则制定者 |
| **summary** | 文档摘要 | 100-200字 |
| **keywords** | 关键词 | 5-10个 |
| **related_docs** | 相关文档ID | 至少2个 |
| **query_patterns** | 查询模式 | 3-5个 |
| **doc_weight** | 文档权重 | core/important/reference |
| **layer** | 文档层级 | dao/shu |
| **training_order** | 培训顺序 | 数字 |

## ✅ 提交前检查清单

### 内容质量
- [ ] 有震撼的故事开头
- [ ] 有明确的核心洞察
- [ ] 有可迁移的思维框架
- [ ] 有3个以上案例（包含失败案例）
- [ ] 有具体可执行的行动建议
- [ ] 有至少3条金句
- [ ] 有至少2篇相关文档引用

### 写作风格
- [ ] 语气符合"带徒弟的CEO"
- [ ] 每句话<40字
- [ ] 有数据支撑（不空洞）
- [ ] 有画面感（不抽象）
- [ ] 无学术八股（接地气）

### 格式规范
- [ ] 标题层级正确（#、##、###）
- [ ] 表格格式正确
- [ ] 文件命名规范

### 索引完整
- [ ] YAML索引已创建
- [ ] doc_id正确（不重复）
- [ ] keywords充分（5-10个）
- [ ] related_docs已填写
- [ ] query_patterns合理

## 🚀 Git提交模板

```bash
# 1. 添加文件
git add [文档路径]
git add rag-index/indexes/[索引文件]

# 2. 提交
git commit -m "📝 新增：[文档标题]

- 分类：[道层/术层] - [子分类]
- 文档ID：[DOC-ID]
- 主题：[核心主题]
- 关键洞察：[一句话核心洞察]"

# 3. 推送
git push origin master

# 4. 更新RAG索引
python scripts/update_index.py DOC-[ID]
```
